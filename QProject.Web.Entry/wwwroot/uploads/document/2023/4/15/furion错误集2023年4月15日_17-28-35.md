## 字符串数组

在c 语言中 属性的返回值为  string[]

```
例如 public string[] name{get; set;}

 xxx.name = new string[]{ "a", "b", "c"} 
 xxx.name[] = {"a", "b", "c"}
 
 错误示例： name = ["a", "b", "c"]
```

  实例化这个类属性时，需new string[] 才可使编译器正确解析字符串数组

## Object reference not set to an instance of an object

```
示例代码
var info = await projectfileInfo
                .Where(a => a.FCID == fcid)
                .Select(a => new ProjectFileInfoDtos
            {
                FIID = a.FIID,
                FlieName = a.FlieName,
                UploadName = a.UploadName,
                Unit = a.Unit,
                IdCardInform =  DESCEncryption.Decrypt(a.IdCardInform.ToString(), "IdCard", false)    //报错原因位置
                }
错误分析：这里使用了空引用检查，如果`a.IdCardInform`是空，则返回`null`，否则进行解密操作。另外，使用`FirstOrDefault()`方法保证返回的是一个对象而不是一个集合，以避免出现`null`异常。
修改： IdCardInform = a.IdCardInform != null ? DESCEncryption.Decrypt(a.IdCardInform.ToString(), "IdCard", false) : null
成功！！！
```

## 显示无法使用实例引用来访问成员，请改用类型名限定它

```
报错代码：DataFormatTools dataFormat = new DataFormatTools();   //将DataFormatTools类实例化
		随后：dataFormat.ReplaceWithSpecialChar(DESCEncryption.Decrypt(a.IdCardInform.ToString(), "IdCard", false).ToString(),1,2,'*')	   //想的是通过实例化的dataFormat调用类DataFormatTools中的方法成员ReplaceWithSpecialChar()
      
 修改后： DataFormatTools.ReplaceWithSpecialChar(DESCEncryption.Decrypt(a.IdCardInform.ToString(), "IdCard", false).ToString(),1,2,'*')	    //DataFormatTools为类名
 原因: 可直接通过类名直接访问类中方法成员
```
